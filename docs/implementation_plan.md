# 📝 페르소나 기반 특허/지식재산권 칼럼 자동 생성기 기획안

이 문서는 5명의 특허법인 변리사(윤웅채, 김신연, 이상담, 김봉근, 백상희)의 고유한 문체와 전문 지식을 모방하여 칼럼을 작성하는 AI 시스템의 개발 계획입니다.

## 1. 개요 및 요구사항
- **목표:** 특정 주제/검색어를 입력하면 선택한 변리사가 직접 쓴 것 같은 고품질 지식재산권 칼럼을 자동 생성합니다.
- **핵심 기능:**
  - 웹 블로그 링크를 통한 변리사별 과거 글 데이터 수집 및 학습 (저의 내장 스크래핑 기능 활용)
  - **특정 조건(예: 2020~2022년 작성 글)에 따른 선별적 스크래핑 및 학습 필터링**
  - 선택한 변리사의 페르소나(문체, 주로 사용하는 어휘, 논리 전개 방식 등)가 반영된 글 생성
  - AI 생성 느낌을 최소화하고, 실제 전문가가 작성한 수준의 결과물 도출

## 2. 시스템 아키텍처 및 기술 스택
이 프로젝트는 생산성과 사용성을 고려하여 빠르고 강력한 **Python 기반의 Streamlit 웹 애플리케이션**으로 구축하는 것을 제안합니다. 사용자의 윈도우 환경 내에서 구동되며, 직관적인 UI를 제공합니다.

- **프론트엔드 (UI):** `Streamlit` (드롭다운, 입력창, 결과 화면 등을 깔끔하게 구성 가능)
- **백엔드 및 로직:** `Python`
- **언어 모델 (LLM):** OpenAI API (GPT-4o) 또는 Anthropic API (Claude 3.5 Sonnet) (문체 모방에 뛰어난 모델 사용 권장)
- **데이터 저장소 (Vector DB):** `ChromaDB` 또는 `FAISS` (각 변리사의 블로그 글을 임베딩하여 로컬에 저장)
- **스크래핑 엔진:** `BeautifulSoup`, `Selenium` 등 (발행일자 추출 및 필터링 적용)
- **RAG 파이프라인:** `LangChain` 활용

### 3.1. Data Ingestion (데이터 수집 및 학습 모듈)
- **기능:** 사용자님께서 각 변리사의 블로그 링크를 제공해 주시면 접속하여 텍스트를 추출합니다. **추가로 날짜/연도 필터링을 설정하여 원하는 기간의 글만 학습 데이터로 선별합니다.**
- **파일:** `[NEW] src/scraper.py`
- **처리:** 추출된 텍스트 중 설정된 기간에 부합하는 글만 필터링한 후, 논리적인 단락으로 분할(Chunking)하고 벡터화하여 변리사별로 독립된 DB에 저장합니다. (Few-shot Prompting용)

### 3.2. Persona Prompt Engine (페르소나 생성 엔진)
- **기능:** 각 변리사의 고유한 성향을 정의하는 프롬프트 템플릿 관리.
- **파일:** `[NEW] src/engine.py` / `[NEW] src/prompts.py`
- **처리:** 
  1. 사용자 입력 (예: "스타트업 상표권 등록의 중요성") 수신
  2. 선택된 변리사의 Vector DB에서 관련 과거 글귀(문체 참고용) 검색
  3. "당신은 특허법인 ○○○의 [변리사 이름] 변리사입니다. 다음 제공된 당신의 과거 글 스타일을 참고하여..." 형태의 정교한 RAG 프롬프트 구성

### 3.3. User Interface (사용자 화면 - Streamlit)
- **사이드바:** API 키 입력 및 설정, 변리사 선택 옵션
- **파일:** `[NEW] app.py` 
- **메인화면:** 
  - 칼럼 주제 및 핵심 키워드 입력창
  - 새로운 블로그 링크를 입력해 해당 변리사의 DB를 업데이트하는 기능
  - '칼럼 생성하기' 버튼 밑 즉시 확인 및 복사 가능한 에디터 영역

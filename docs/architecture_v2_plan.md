# 🏗️ 특허법인 AI 칼럼 자동 생성기 V2 고도화 기획안

현재 완성된 V1은 단일 프롬프트(`engine.py` 내 하드코딩)와 단일 데이터 파일(`all_texts_compiled.json`)에 의존하고 있습니다. 출력물의 퀄리티를 지속적으로 높이고 유지보수를 쉽게 만들기 위해, **설정 기반(Configuration-driven) 및 모듈화된 아키텍처(V2)**로 개편하는 것을 제안합니다.

## 🎯 핵심 목표 (요구사항 해결)
1. **출력 포맷의 유연한 수정**: 파이썬 코드를 수정하지 않고 텍스트/마크다운 템플릿 파일만 수정하여 글의 구조(인사말-본론-결론 형식 등)를 변경.
2. **손쉬운 페르소나 DB 관리**: 특정 변리사 폴더에 텍스트 파일이나 링크 목록명만 던져넣으면 자동으로 최신 학습 데이터로 편입.
3. **변리사별 맞춤형 프롬프트 적용**: 공통 프롬프트 외에, 각 변리사에게만 적용되는 특별 지시사항(예: "백상희 변리사는 항상 판례 번호를 굵은 글씨로 강조합니다")을 독립적으로 관리.

## 📂 제안하는 V2 디렉토리 구조

데이터와 프롬프트를 코드(`src/`)에서 완전히 분리하여, **개발 지식이 없어도 파일만 수정하면 AI가 똑똑해지도록** 구조를 변경합니다.

```text
column_generator/
│
├── config/                 # ⚙️ 프롬프트 및 포맷 템플릿 보관소 (코드 수정 없이 여기서 관리)
│   ├── base_prompt.md      # 시스템 전체 공통 지침 및 기본 포맷 형태
│   └── personas/           # 개별 변리사 맞춤형 프롬프트 (오버라이딩)
│       ├── yun_ung_chae.json  # 예: {"tone": "강력하고 단호함", "special_rule": "판례 강조"}
│       └── kim_sin_yeon.json
│
├── persona_db/             # 📚 변리사별 지식/문체 DB 보관소
│   ├── yun_ung_chae/
│   │   ├── blogs.json      # 스크래핑된 기존 블로그 글 
│   │   ├── extra_text.txt  # (추가 기능) 사용자가 복붙해 넣은 참고용 판례나 텍스트
│   │   └── links.txt       # (추가 기능) 스크래퍼가 긁어올 타겟 URL 목록
│   └── kim_sin_yeon/
│       └── ...
│
├── src/                    # 💻 코딩된 파이썬 기능 구현부
│   ├── engine.py           # config/ 및 persona_db/ 의 데이터를 합쳐서 Claude에 전송
│   ├── scraper.py          # persona_db/*/links.txt를 읽어 자동으로 긁어오는 로직
│   └── ...
└── app.py                  # Streamlit UI
```

## 🛠️ 기능별 상세 구현 방안

### 1. 출력되는 글의 포맷 수정 (Output Format)
- **현재**: `engine.py`의 `BASE_PROMPT` 변수 안에 길게 텍스트로 박혀 있습니다.
- **개선안**: `config/base_prompt.md` 마크다운 파일로 분리합니다.
  - 마케팅 담당자나 사용자가 단순히 메모장으로 열어서 *"3. 결론 파트에서는 항상 무료 상담 소제목을 추가해라"* 와 같이 자연어로 수정 및 저장하면, 다음 번 생성부터 즉시 포맷이 반영됩니다.

### 2. 특정 변리사의 DB (파일, 텍스트, 링크) 추가
- **현재**: 코드를 실행하여 스크래퍼가 만든 하나의 json 덩어리만 봅니다.
- **개선안**: `persona_db/{변리사_이름}/` 독립 폴더 시스템 도입.
  - **텍스트 추가**: 변리사님이 쓰신 좋은 외부 기사나 문서가 있다면, `extra_text.txt` 파일에 텍스트를 복사-붙여넣기만 하면 됩니다.
  - **링크 추가**: 스크래핑하고 싶은 특정 블로그 주소들을 `links.txt` 파일에 줄바꿈으로 추가해두고, UI에서 **"DB 업데이트 버튼"**을 누르면 자동으로 스크래퍼가 해당 링크들을 긁어와 `blogs.json`에 합칩니다.
  - 엔진(`engine.py`)은 해당 폴더 안의 모든 텍스트/JSON 문서를 긁어 모아 Context로 주입합니다.

### 3. 특정 변리사에게만 작동하는 프롬프트 개선 (Persona Overrides)
- **현재**: 모든 변리사가 동일한 `BASE_PROMPT` 뼈대(이름만 치환)를 공유합니다.
- **개선안**: `config/personas/{변리사_이름}.json` 파일 도입.
  - 공통적인 포맷 지시어(`base_prompt.md`)는 유지하되, 특정 변리사를 선택했을 때 이 json 파일의 내용이 **"추가 지시사항"**으로 Claude 모델에게 덧붙여집니다.
  - **예시 구성 (`김봉근.json`)**:
    ```json
    {
      "personality": "전문적인 용어를 많이 쓰지만 비유를 들어 설명함.",
      "strict_rules": [
         "반드시 글의 서두에 최근 경제 뉴스를 인용할 것",
         "맺음말에는 '특허법인 테헤란 김봉근이었습니다'를 쓸 것"
      ]
    }
    ```
  - 누군가 문체를 보완하고 싶다면 코드 개편 없이 이 JSON 파일의 텍스트만 덧붙이면 끝납니다.

## 🚀 다음 단계 (Action Plan)

이 구조가 마음에 드신다면, 다음 순서로 즉각 업그레이드 작업을 진행할 수 있습니다.

1. **Config 및 DB 폴더 구조 생성**: 기존 하드코딩된 프롬프트를 `config/base_prompt.md`로 빼내는 작업.
2. **엔진(`engine.py`) 리팩토링**: 
   - `base_prompt.md`와 선택된 `persona.json` 파일을 합쳐서 동적으로 프롬프트를 조립하는 로직 개발.
   - `persona_db/{id}/` 내의 여러 종류 파일(txt, json)을 모두 읽어와 Context를 풍부하게 만드는 기능 탑재.
3. **UI(`app.py`) 업데이트**: (사용자 요청 시) 관리자가 Streamlit 화면 내에서 직접 프롬프트를 텍스트창으로 수정하거나, 텍스트를 복붙하여 DB 폴더에 즉시 저장하는 [관리자 모드 탭] 추가. 

이 기획안의 방향성이 원하시는 고도화 방향과 일치하는지 확인해주시면, 즉시 1번 단계부터 작업을 시작하겠습니다!

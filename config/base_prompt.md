# 출력 규칙

1. 제목은 포함하지 않고 본문만 바로 시작할 것
2. "네, 작성해보겠습니다" 같은 AI 서론 절대 금지. 곧바로 칼럼 본문 시작.
3. 길이: 블로그 포스팅에 적합하게 1500자~3500자 (너무 짧아도, 너무 길어도 안 됨)
4. 법적/기술적 내용은 실제 한국 특허법 및 실무에 부합하게 정확히 작성
5. 모바일 가독성 고려: 한 문장은 50자 이내 권장, 문단 사이 빈 줄 필수
6. 소제목은 2~4개, 각 소제목 아래 내용 길이는 의도적으로 불균형하게
7. CTA는 공지글 링크 방식으로 자연스럽게 (전화번호 직접 노출 금지)
8. 과거 글과 문장 단위 유사도가 높으면 안 됨. 같은 표현/사례를 반복하지 말 것.

# 마크다운 형식 규칙 (필수)

아래 형식을 반드시 준수해야 블로그에 올바르게 포맷팅됩니다.

1. 소제목은 반드시 `## ` (샵 두 개 + 공백)으로 시작할 것
   - 예시: `## 상표등록, 왜 지금 해야 하는가`
   - ❌ 틀린 예: `상표등록, 왜 지금 해야 하는가` (## 없이 쓰면 소제목 처리 안 됨)
   - ❌ 틀린 예: `**상표등록, 왜 지금 해야 하는가**` (볼드로만 쓰면 소제목 아님)
2. 소제목 텍스트는 짧고 임팩트 있게 (10~30자 이내)
3. 강조 키워드는 `**키워드**` (볼드) 사용
4. 인용/명언/핵심 문장은 `"큰따옴표"`로 감쌀 것 (블로그에서 인용구 박스로 표시됨)
5. 내부 링크 URL은 단독 줄에 `https://...` 형태로 삽입
6. 리스트는 `- 항목` 형태로 작성
